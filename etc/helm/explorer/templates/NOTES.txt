Your Studio server has now been installed, and can be accessed in the following ways:

1. Connect to the Studio CLI pod:

  kubectl exec -n {{ .Release.Namespace }} $(kubectl get pod -n {{ .Release.Namespace }} --selector=app={{ template "explorer.name" . }} -o jsonpath='{.items...metadata.name}') -- sh

2. Connect with HTTP internally, within the explorernetes cluster on:

  {{ template "explorer.service-name" . }}.{{ .Release.Namespace }}:{{ .Values.explorer.port }}

3. Connect with HTTP Node Port to the explorernetes cluster:

  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ template "explorer.service-name" . }})
  echo http://$NODE_IP:$NODE_PORT/mesh

4. Connect with HTTP external service to the explorernetes cluster:

  export POD_NAME=$(kubectl get pods --namespace {{ .Release.Namespace }} -l "app={{ template "explorer.name" . }}" -o jsonpath="{ .items[0].metadata.name }")
  echo "Visit http://127.0.0.1:{{ .Values.explorer.port }}/mesh to access Studio"
  kubectl port-forward --namespace {{ .Release.Namespace }} $POD_NAME {{ .Values.explorer.port }}:{{ .Values.explorer.port }}

5. For Minikube:

  open http://minikube.datalayer.io.local/mesh
